① Đảm bảo Entity có Decorator

Mỗi field trên form = 1 property có decorator trên Entity.

@TableDecorator({ title: 'Category' })
export class CategoryEntity extends BaseEntity {
  @StringDecorator({ required: true, max: 20 })
  Code: string;
  @StringDecorator({ required: true, max: 200 })
  Name: string;
  @StringDecorator({ type: StringType.MultiText })
  Description: string;
}

📄 edit.category.component.ts
@Component({ templateUrl: './edit.category.component.html',
  styleUrls: ['./edit.category.component.scss', 'assets/css/modal.scss'] })
export class EditCategoryComponent extends EditComponent implements OnInit {
  id: number;  loading = true;
  item = new CategoryEntity();
  constructor() { super(); this.state = this.getUrlState(); }

  async ngOnInit() {
    this.id = this.getParam('id');  this.viewer = this.getParam('viewer');
    if (this.state) this.addBreadcrumb(this.id ? 'Sửa' : 'Thêm');
    this.renderActions();
    await this.loadItem();  this.loading = false;
  }
  private async loadItem() {
    this.item = new CategoryEntity();
    if (this.id)
      await this.service.item('category', this.id).then(r => {
        if (ResultApi.IsSuccess(r))
          this.item = EntityHelper.createEntity(CategoryEntity, r.Object);
      });
  }
  private renderActions() {
    this.actions = this.id
      ? [ActionData.back(() => this.back()), ActionData.saveUpdate('Lưu', () => this.confirmAndBack())]
      : [ActionData.back(() => this.back()), ActionData.saveAddNew('Thêm', () => this.confirmAndBack())];
  }
  private async confirmAndBack() {
    if (await this.autoConfirm(this.item)) this.back();
  }
}

⊡ <editor> Component

Tự render input theo decorator trên Entity. Không cần viết HTML thủ công.

AttributeTypeMô tả
[(object)]EntityBắt buộc. Two-way binding. VD: [(object)]="item"
propertystringBắt buộc. Tên property. VD: property="Name"
[decorator]objectOverride runtime. VD: [decorator]="{ readonly: viewer }"

💡 Ví dụ

<!-- Cơ bản -->
<editor [(object)]="item" property="Name"></editor>

<!-- Override readonly -->
<editor [(object)]="item" property="Title"
  [decorator]="{ readonly: viewer, label: '' }"></editor>

<!-- Layout Bootstrap col -->
<div class="row">
  <div class="col-sm-6">
    <editor [(object)]="item" property="Code"></editor>
  </div>
  <div class="col-sm-6">
    <editor [(object)]="item" property="Name"></editor>
  </div>
</div>

✓ Validation

Import từ core/decorators/validator. Validation tự động chạy dựa trên decorator (required, max, min, validators).

📦 Functions
FunctionMô tả
validation(item, cols?)Validate 1 item. Trả về Promise<boolean>. Tự emit lỗi lên <editor>.
validations(items[], cols?)Validate mảng items. Dừng ở item đầu tiên lỗi (hoặc multiple=true để check hết).
clearValidation(item)Xoá tất cả lỗi validation hiện tại.
catchValidation(item, prop, msg)Gán lỗi thủ công cho 1 property. VD: lỗi từ server.
🔑 PatternType (validators)

Khai báo: validators: [{ pattern: PatternType.XXX, message?: 'Custom msg' }]

PatternTypeÁp dụng choMô tả
RequiredAllTự động khi required: true. Không cần khai báo thủ công.
MinString, Number, DateTime, Image/FileString: min ký tự. Number: min giá trị. DateTime: min date. File: min số file.
MaxString, Number, DateTime, Image/FileTương tự Min nhưng giới hạn trên.
RequiredMatchStringSo sánh 2 field giống nhau. VD: Password + ConfirmPassword.
RequiredMinNumber, DateTime, DropdownGiá trị phải >= field khác. VD: EndDate >= StartDate.
RequiredMaxNumber, DateTime, DropdownGiá trị phải <= field khác.
UniqueStringCheck trùng qua API. Cần unique: { url, property, propertyId? }.
DuplicateStringKhông trùng với field khác trong cùng entity. Cần duplicates: ['FieldB'].
CardNumberStringValidate số thẻ tín dụng (Luhn).

💡 Ví dụ

// 1. autoConfirm — validate + save tự động
if (await this.autoConfirm(this.item)) this.back();

// 2. Validate thủ công
if (await validation(this.item)) { /* save logic */ }

// 3. Validate chỉ 1 số cột
if (await validation(this.item, ['Code', 'Name'])) { ... }

// 4. Gán lỗi custom (VD: lỗi từ API)
catchValidation(this.item, 'Email', 'Email đã tồn tại');

// 5. Xoá lỗi
clearValidation(this.item);

🪟 Popup — Mở form trong popup

Có 3 hàm popup chính: popupAsync (custom component), popupAutoAsync (auto-form từ Entity), popupSubGrid (sub-grid popup).

1️⃣ popupAsync — Popup Component tùy chỉnh

Mở popup chứa một component Angular tùy chỉnh. Wrapper của WapperAsync.

this.popupAsync({
  object: EditCategoryComponent,
  title: 'Chỉnh sửa danh mục',
  size: ModalSizeType.Large,
  objectExtra: { id: item.Id },
  confirmText: 'Lưu',
}, async (result) => {
  // callback khi bấm nút Lưu
  await this.loadItems();
});
ParamTypeMô tả
objectComponentAngular Component để render trong popup.
objectExtraanyDữ liệu truyền vào component (truy cập qua getParamPopup('key')).
titlestringTiêu đề popup.
sizeModalSizeTypeSmall, Medium, Large, ExtraLarge, FullScreen.
confirmTextstringText nút xác nhận. Nếu null → ẩn nút.
cancelTextstringText nút đóng. Mặc định: 'Đóng'.
confirmClosebooleanTự đóng popup sau khi confirm.
2️⃣ popupAutoAsync — Auto-form từ Entity

Tự sinh form popup từ Entity reference + properties. Không cần viết component riêng.

this.popupAutoAsync({
  title: 'Thêm danh mục',
  size: ModalSizeType.Medium,
  objectValue: {
    reference: CategoryEntity,
    properties: ['Code', 'Name', 'Description'],
    object: new CategoryEntity(),  // giá trị mặc định
  },
  confirmText: 'Lưu',
  confirmFunction: async (item) => {
    // item = entity object với dữ liệu user đã nhập
    await this.service.save('category', item);
  },
});
Param (DialogAutoData)TypeMô tả
objectValue.referenceEntity classEntity dùng để tự sinh form (đọc decorator).
objectValue.properties(string | ObjectEx)[]Danh sách properties hiển thị trên form.
objectValue.objectanyGiá trị mặc định cho form (entity instance).
confirmFunction(item) => PromiseCallback nhận entity object khi bấm xác nhận.
title / sizestring / enumTiêu đề và kích thước popup.
ℹ️Khác biệt: popupAsync cần viết component riêng, popupAutoAsync tự sinh form từ Entity decorators → nhanh hơn cho form đơn giản.
