🔗 Column — Truy vấn dữ liệu bảng liên kết (Join)

Khi khai báo Column, grid sẽ gửi đến API dạng "Property=Column". API dùng Dynamic LINQ Select để truy vấn dữ liệu từ bảng liên kết qua EF Navigation Property.

📐 Cú pháp
DạngColumn valueKết quả SQL
Navigation PropertySmtpAccount.UserNameJOIN SmtpAccount → SELECT SmtpAccount.UserName as SmtpAccountId
Count()StaffMembers.Count()SELECT (SELECT COUNT(*) FROM StaffMembers ...) as MemberCount
Sum()Deals.Sum(d => d.Amount)SELECT (SELECT SUM(Amount) FROM Deals ...) as TotalAmount
Any()Tasks.Any()SELECT EXISTS(...) as HasTasks
💡 Ví dụ
// 1. Hiển thị UserName từ bảng SmtpAccount (FK: SmtpAccountId)
{ Property: 'SmtpAccountId', Column: 'SmtpAccount.UserName' }
// → API nhận: "SmtpAccountId=SmtpAccount.UserName"
// → LINQ Select: SmtpAccount.UserName as SmtpAccountId

// 2. Đếm số member trong team
{ Property: 'MemberCount', Column: 'StaffMembers.Count()' }
// → API nhận: "MemberCount=StaffMembers.Count()"

// 3. Tổng giá trị đơn hàng
{ Property: 'TotalAmount', Column: 'Orders.Sum(o => o.Amount)' }
🔄 Luồng xử lý
① Frontend: getColumns()
   Properties → ["Id", "Name", "SmtpAccountId=SmtpAccount.UserName"]

② Gửi API: POST /admin/{controller}/Items
   TableData.Columns = ["Id", "Name", "SmtpAccountId=SmtpAccount.UserName"]

③ Backend: GetDynamicQuery(columns)
   query.Select("new (Id as Id, Name as Name, SmtpAccount.UserName as SmtpAccountId)")
ℹ️Auto columns: API tự thêm IsActive, IsDelete, CreatedDate, UpdatedDate, CreatedBy=CreatedByUser.UserName, UpdatedBy=UpdatedByUser.UserName nếu chưa có.
⚠️Column phải trùng với EF Navigation Property đã cấu hình trong Entity. Nếu không có navigation → API sẽ lỗi.

🔍 Filters — Bộ lọc

3 loại filter khác nhau về UI và cách hoạt động.

1️⃣ InlineFilters — Lọc cùng hàng tìm kiếm

Hiển thị ngay bên cạnh ô tìm kiếm, thay đổi → tự filter.

InlineFilters: ['OrganizationId']
// → render dropdown "Organization" cùng hàng search box
TypeKhai báoMô tả
stringProperty nameTự tạo editor từ decorator của property đó.
GridFilterDataObject{ Name, LookupUrl?, Multiple?, AllowClear?, Types? }
2️⃣ CustomFilters — Bộ lọc nâng cao

Ẩn mặc định, bấm nút "Bộ lọc" trên toolbar để mở ra panel filter với nhiều trường.

CustomFilters: ['Status', 'CreatedDate', 'CategoryId']
// → panel filter với 3 trường: Status, CreatedDate, CategoryId
3️⃣ TabFilters — Lọc dạng tab

Tạo các tab phía trên grid. Mỗi tab = 1 giá trị filter. Hiện count theo tab.

TabFilters: [
  { Name: 'Status', Title: 'Chờ',       Value: 0, CompareType: CompareType.N_Equals },
  { Name: 'Status', Title: 'Đang xử lý', Value: 1, CompareType: CompareType.N_Equals },
  { Name: 'Status', Title: 'Đã xử lý',  Value: 2, CompareType: CompareType.N_Equals },
]
FieldTypeMô tả
NamestringTên property filter.
TitlestringText hiển thị trên tab.
ValueanyGiá trị filter khi tab active.
CompareTypeCompareTypeKiểu so sánh.
IconstringIcon cho tab.
CountnumberSố lượng (tự cập nhật từ API).
ActivebooleanTab mặc định active.

TabChange() => anyCallback khi chuyển tab.

① Entity

Tạo Entity class với @TableDecorator và các field decorator. Grid sẽ tự đọc metadata từ đây.

📄 category.entity.ts
@TableDecorator({ name: 'category', title: 'Danh mục' })
export class CategoryEntity extends BaseEntity {
  @StringDecorator({ label: 'Tên', required: true, max: 150 })
  Name: string;
  @StringDecorator({ label: 'Mã', max: 50 })
  Code: string;
  @BooleanDecorator({ label: 'Kích hoạt' })
  IsActive: boolean;
}
ℹ️@TableDecorator.name sẽ tự động trở thành ReferenceName (tên controller API).
② Grid Component

Tạo component kế thừa GridComponent. Khai báo obj: GridData và gọi this.render(obj) trong constructor.

📄 Đơn giản — category.module.ts
@Component({ templateUrl: '../../../core/components/grid/grid.component.html' })
export class CategoryComponent extends GridComponent {
  obj: GridData = {
    Reference: CategoryEntity,
    Size: ModalSizeType.FullScreen,
    Properties: ['Id', 'Name', 'Code', 'IsActive'],
  };
  constructor() { super(); this.render(this.obj); }
}
📄 Nâng cao — custom properties + actions
@Component({ templateUrl: '../../../core/components/grid/grid.component.html' })
export class TeamComponent extends GridComponent {
  obj: GridData = {
    Reference: TeamEntity,
    Navigation: true,
    InlineFilters: ['OrganizationId'],
    Actions: [
      ActionData.edit((item) => this.edit(item)),
      ActionData.delete((item) => this.trash(item)),
    ],
    MoreActions: [
      ActionData.history((item) => this.viewHistory(item.Id)),
    ],
  };
  constructor() {
    super();
    this.properties = [
      { Property: 'Id', Type: DataType.Number },
      { Property: 'Name', Format: (item) => '<a>' + item.Name + '</a>' },
      { Property: 'Code' },
      { Property: 'Amount', Align: 'center',
        Click: (item) => this.popup(item),
        Format: (item) => item.Amount + ' nhân viên' },
    ];
    this.render(this.obj);
  }
}
⚠️this.render(this.obj) phải gọi sau khi set this.properties. Nếu khai báo Properties trực tiếp trong obj thì không cần set riêng.
③ Đăng ký Module + Routes

Khai báo component trong NgModule và đăng ký các routes.

📄 category.module.ts (phần @NgModule)
@NgModule({
  declarations: [CategoryComponent, EditCategoryComponent],
  imports: [
    UtilityModule,
    RouterModule.forChild([
      { path: '', component: CategoryComponent, canActivate: [AdminAuthGuard] },
      { path: 'add', component: EditCategoryComponent, canActivate: [AdminAuthGuard] },
      { path: 'edit', component: EditCategoryComponent, canActivate: [AdminAuthGuard] },
      { path: 'view', component: EditCategoryComponent, canActivate: [AdminAuthGuard] },
    ])
  ]
})
export class CategoryModule { }
ℹ️Nếu grid dùng Navigation: true, cần override addNew(), edit(item), view(item) với NavigationStateData.
📄 addNew / edit / view (khi Navigation = true)
addNew() {
  let obj: NavigationStateData = { prevData: this.itemData, prevUrl: '/admin/category' };
  this.router.navigate(['/admin/category/add'], { state: { params: JSON.stringify(obj) } });
}
edit(item: CategoryEntity) {
  let obj: NavigationStateData = { id: item.Id, prevData: this.itemData, prevUrl: '/admin/category' };
  this.router.navigate(['/admin/category/edit'], { state: { params: JSON.stringify(obj) } });
}
view(item: CategoryEntity) {
  let obj: NavigationStateData = { id: item.Id, viewer: true, prevData: this.itemData, prevUrl: '/admin/category' };
  this.router.navigate(['/admin/category/view'], { state: { params: JSON.stringify(obj) } });
}

🪟 Popup — Mở popup từ Grid

GridComponent có 4 hàm popup. Khác với Forms, Grid tự tối ưu popup (ẩn toolbar, set IsPopup).

1️⃣ popupAsync — Popup Component tùy chỉnh

Mở popup chứa Angular Component. Truyền data qua objectExtra.

this.popupAsync({
  object: EditCategoryComponent,
  title: 'Chỉnh sửa',
  size: ModalSizeType.Large,
  objectExtra: { id: item.Id },
  confirmText: 'Lưu',
}, async (result) => {
  await this.loadItems();
});
Param (DialogData)TypeMô tả
objectComponentAngular Component render trong popup.
objectExtraanyData truyền vào component → this.getParamPopup('key').
titlestringTiêu đề popup.
sizeModalSizeTypeSmall · Medium · Large · ExtraLarge · FullScreen
confirmTextstringText nút xác nhận. null → ẩn nút.
cancelTextstringText nút đóng (mặc định: 'Đóng').
confirmClosebooleanTự đóng popup sau confirm.
descriptionstringMô tả phụ dưới tiêu đề.
2️⃣ popupAboveAsync — Popup lớp 2 (chồng lên popup cũ)

Mở popup thứ 2 phía trên popup hiện tại. Cùng API với popupAsync.

// Từ trong popup 1, mở popup 2
this.popupAboveAsync({
  object: ChoiceUserComponent,
  title: 'Chọn nhân viên',
  size: ModalSizeType.Large,
  confirmText: 'Chọn',
}, async (selectedUsers) => {
  // xử lý users đã chọn
});
3️⃣ popupSubGrid — Sub-grid trong popup

Mở 1 grid con trong popup. Grid tự ẩn toolbar (Exports/Imports/Filters/Actions).

this.popupSubGrid({
  Title: 'Danh sách nhân viên',
  Reference: StaffEntity,
  Size: ModalSizeType.ExtraLarge,
  Properties: ['Id', 'Name', 'Email', 'Phone'],
  Checkable: true,
}, existingItems, async (selectedItems) => {
  // selectedItems = mảng items được check
  await this.saveMembers(selectedItems);
});
ℹ️ Auto-config: popupSubGrid tự set: IsPopup = true, ẩn Actions, Imports, Exports, Filters, Features, HideHeadActions = true. Chỉ cần khai báo Title, Reference, Properties.
ParamTypeMô tả
obj (arg 1)GridDataConfig grid con. Tự ẩn toolbar nếu không khai báo.
items (arg 2)BaseEntity[]Dữ liệu sẵn có (preload). Nếu null → grid tự fetch từ API.
okFunction (arg 3)(items) => PromiseCallback khi bấm xác nhận, nhận mảng items (checked nếu Checkable).
⚠️Nếu muốn giữ CustomFilters → khai báo CustomFilters trong obj. Nếu không → tự ẩn (HideCustomFilter = true).
4️⃣ popupAutoAsync — Auto-form từ Entity

Tự sinh form popup từ Entity decorators. Không cần viết component riêng.

this.popupAutoAsync({
  title: 'Thêm nhanh',
  size: ModalSizeType.Medium,
  objectValue: {
    reference: CategoryEntity,
    properties: ['Code', 'Name', 'Description'],
    object: new CategoryEntity(),
  },
  confirmText: 'Lưu',
  confirmFunction: async (item) => {
    await this.service.save('category', item);
    await this.loadItems();
  },
});
Param (DialogAutoData)TypeMô tả
objectValue.referenceEntity classEntity dùng để tự sinh form (đọc decorator).
objectValue.properties(string | ObjectEx)[]Properties hiển thị trên form.
objectValue.objectanyGiá trị mặc định (entity instance).
confirmFunction(item) => PromiseCallback nhận entity khi bấm xác nhận.
ℹ️So sánh: popupAsync = popup với component tùy chỉnh (full control), popupAutoAsync = auto-form nhanh, popupSubGrid = grid con pick items.
